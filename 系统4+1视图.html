AutoTestWeb - 芯片验证防护网系统

4+1视图架构模型

AutoTestWeb是一个针对芯片验证的自动化测试平台，提供了环境管理、测试套管理、任务执行、结果跟踪等全流程功能，帮助开发团队高效地进行芯片验证工作。

1. 场景视图 (Scenario View)

场景视图描述了系统的主要功能场景和用户交互流程，通过典型用例展示系统如何满足用户需求。

典型场景：执行自动化测试任务

参与者：测试工程师

目标：创建并执行一个自动化测试任务，查看测试结果

  1. 测试工程师登录系统
  2. 在环境管理模块确认测试环境已配置
  3. 在测试套管理模块选择合适的测试套
  4. 进入新增执行任务页面，填写任务信息（选择测试套和环境）
  5. 提交任务并在任务列表查看任务执行状态
  6. 任务执行完成后，查看任务详情和测试报告

其他重要场景：

  • 管理测试环境配置
  • 创建和维护测试用例集
  • 查看和分析历史测试结果
  • 管理系统用户和权限

2. 逻辑视图 (Logical View)

逻辑视图描述了系统的功能模块划分和它们之间的逻辑关系，关注系统提供的功能和服务。

模块功能说明：

  • 用户认证与授权：提供用户登录、注册、权限管理等功能，基于Django内置认证系统实现
  • 环境管理：管理测试环境的配置信息，支持环境的增删改查
  • 测试套管理：组织和管理测试用例集合，支持测试套的创建和维护
  • 任务执行管理：负责测试任务的创建、调度和执行状态跟踪
  • 结果管理：收集和存储测试结果，提供结果查询和统计分析功能
  • 模块管理：管理系统中的功能模块信息
  • 特性测试用例管理：管理针对特定特性的测试用例

3. 进程视图 (Process View)

进程视图描述了系统的运行时进程和它们之间的交互方式，关注系统的动态行为和性能特性。

前端进程

基于Vue 3的单页应用(SPA)，在浏览器中运行，负责用户界面渲染和交互处理。

后端API服务进程

Django应用服务，处理HTTP请求，提供RESTful API接口。

  • 处理用户认证和请求鉴权
  • 执行业务逻辑处理
  • 与数据库交互
  • 返回处理结果

数据库进程

SQLite数据库服务，存储系统数据。

  • 用户信息存储
  • 测试环境配置存储
  • 测试套和测试用例数据存储
  • 任务执行记录和结果存储

任务执行进程

负责实际执行测试任务的后台进程，可能基于Celery实现异步任务处理。

生产环境部署建议：

  • 前端：使用Nginx静态资源服务器部署编译后的前端代码
  • 后端：使用Gunicorn/uWSGI作为WSGI服务器，配合Nginx反向代理
  • 数据库：迁移至MySQL或PostgreSQL等生产级数据库
  • 缓存：引入Redis缓存层提高性能
  • 任务队列：使用Celery和RabbitMQ/RabbitMQ处理异步任务

5. 开发视图 (Development View)

开发视图描述了系统的代码组织结构和开发环境配置，关注开发团队的工作方式和工具。

项目结构

前端 (frontend/)

  • src/ - 源代码目录
  • src/views/ - Vue组件
  • src/router/ - 路由配置
  • src/api/ - API接口封装
  • tests/ - 前端测试

后端 (backend/)

  • AutoTestWeb/ - 主项目配置
  • common/ - 公共组件
  • env_manager/ - 环境管理模块
  • test_suite/ - 测试套管理模块
  • execution_manager/ - 任务执行管理模块
  • result_manager/ - 结果管理模块

技术栈

  • 前端：Vue 3, Element Plus, Vue Router, Axios
  • 后端：Python, Django, Django REST Framework
  • 数据库：SQLite (开发环境)
  • 开发工具：Vite, Vitest, Jest
  • 版本控制：Git

开发流程

  • 需求分析与功能设计
  • 前后端接口定义
  • 并行开发（前端组件开发，后端API实现）
  • 单元测试与集成测试
  • 代码审查与合并
  • 部署与验证

总结

AutoTestWeb系统采用前后端分离的架构设计，通过Vue 3前端框架和Django后端框架构建了一个功能完整的芯片验证自动化测试平台。系统具有良好的模块化设计，各功能模块职责明确，便于维护和扩展。

通过4+1视图模型的分析，我们全面了解了系统的功能需求、架构设计、运行时行为、部署方式和开发组织，为系统的后续发展和优化提供了清晰的指导。
